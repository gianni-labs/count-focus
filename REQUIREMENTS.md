# Test Countdown - Requerimiento Básico

## Idea

Crear una pequeña aplicación de countdown para terminal, inspirada en `antonmedv/countdown`, pero con una sensación más moderna.

Este proyecto es solo para probar cómo trabaja Pi como agente de programación. No tiene que ser una aplicación profesional ni completa desde el primer intento.

## MVP

Construir una primera versión funcional que permita:

- Ejecutar un countdown desde la terminal.
- Recibir una duración como argumento, por ejemplo `10s`, `5m` o `1h`.
- Mostrar el tiempo restante de forma clara.
- Actualizar el tiempo mientras corre.
- Permitir salir con una tecla como `q`, `Esc` o `Ctrl+C`.
- Incluir una ayuda básica de uso.

## Preferencias

- Preferir Go, salvo que Pi proponga una buena razón para usar otra tecnología.
- Si tiene sentido, usar una librería TUI para que se vea mejor que un simple print en consola.
- Mantener el código simple y fácil de entender.
- Agregar algunos tests si hay lógica que valga la pena testear.

## Ideas Para Después

Estas ideas no son obligatorias para el MVP:

- Pausar y reanudar.
- Modo count-up.
- Título personalizado.
- Barra de progreso.
- Presets como pomodoro.
- Sonido o notificación al terminar.
- Mejor diseño visual en terminal.
- README más completo.
